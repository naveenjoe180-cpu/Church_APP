# Two-Week Prototype Roadmap

## Week 1

### Day 1 to Day 2

- Finalize product scope and UX structure
- Set up Expo app and repository structure
- Prepare Firebase project layout and environment plan

### Day 3 to Day 4

- Build guest-mode home
- Add public church information module
- Add church selector and profile form mockup

### Day 5 to Day 7

- Implement sign-in flow with Google and email/password
- Store profile and church selection in Firestore
- Create pending approval request flow

## Week 2

### Day 8 to Day 9

- Build admin web screens for request review
- Add role assignment and church/team mapping
- Add private dashboard shell after approval

### Day 10 to Day 11

- Build announcements, events, and document modules
- Add team schedule and volunteer acceptance flow for supported teams

### Day 12 to Day 13

- Add anonymous prayer request submission and moderation
- Add notification trigger design

### Day 14

- Test guest flow, approval flow, and role visibility
- Prepare demo build for web and mobile review

## Phase 2 after prototype

- Add OTP verification after approval
- Add multilingual content support
- Add richer admin analytics and church network reporting
- Add stronger scheduling workflows for non-worship teams
